	<title>Azure SQL Database With Direct Connect</title>	<div>

</div><p lang="en-US" style="color: rgb(51, 51, 51); font-family: Calibri; font-size: 10.5pt; margin-top: 22pt; margin-bottom: 0pt;"><span style="font-family: Calibri; font-size: 11pt; font-style: normal; font-weight: normal;">Azure SQL Database with Direct Connect allows you to create dynamic reports based on data and metric you already have in your Azure SQL Database. With Direct Connect, queries are sent back to your Azure SQL Database as you explore the data in the report view. This experience is suggested for users who are familiar with the databases and entities they connect to.</span></p><p lang="en-US" style="color: rgb(51, 51, 51); font-family: Calibri; font-size: 10.5pt; margin-top: 22pt; margin-bottom: 0pt;">Notes: </p><ul><li>Specify the fully qualified server name when connecting (see below for more details)</li><li>Ensure firewall rules for the database are configured to "<a href="https://msdn.microsoft.com/en-us/library/azure/ee621782.aspx">Allow access to Azure services</a>".</li><li>Every action such as selecting a column or adding a filter will send a query back to the database </li><li>Tiles are refreshed approximately every 15 mins (refresh does not need to be scheduled)</li><li>Q&amp;A is not available for Direct Connect datasets</li><li>Schema changes are not picked up automatically</li></ul><p lang="en-US" style="color: rgb(51, 51, 51); font-family: Calibri; font-size: 10.5pt; margin-top: 22pt; margin-bottom: 0pt;"><font color="#333333" face="Calibri" size="3">These restrictions and notes may change as we continue to improve the experiences.
